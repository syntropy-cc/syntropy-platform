# Tutorial: Create an Institution and Project

> **What you'll learn**: Create a digital institution with governance from a template, then add a project and your first issue so others can contribute.
>
> **Prerequisites**: Signed-in user; no prior Hub experience required.
>
> **Time**: ~25 minutes

## Overview

In the Hub, a **digital institution** is the top-level organization: it has a governance contract, members, and projects. In this tutorial you create an institution using a template, then create one project and one open issue so the project is ready for contributions.

## Before You Begin

- [ ] You are signed in to the platform
- [ ] You have access to the Hub (e.g. `https://app.syntropy.cc/hub` or your instance)

## Step 1: Start institution creation

In the Hub, click **Create institution** (or equivalent). You will be asked to choose a **template** (e.g. Personal Institution, Technical Cooperative, Research Laboratory, Collaborative Product, or Advanced).

Choose a template that matches your intent (e.g. **Technical Cooperative** for an open-source-style project).

**Result**: The next screen shows what the template defines (governance rules, roles, etc.) in plain language.

## Step 2: Configure and confirm the institution

Review the template’s parameters. Change any you need (name, founding members, visibility). When ready, confirm.

The system creates the institution, generates the governance contract, and records the founding state in the legitimacy chain.

**Result**: The institution exists; you are a founder and can create projects.

## Step 3: Create the first project

Inside the institution, click **Create project**. Set:

- **Name** and short description
- **Visibility** (public or private)
- Any artifact manifesto or monetization defaults if the UI exposes them

Create the project.

**Result**: The project appears under the institution; you can add issues and manage contributions.

## Step 4: Create the first issue

Open the project and create an **Issue**:

- **Title** — short summary (e.g. “Add input validation to login form”)
- **Description** — what needs to be done, in markdown
- **Acceptance criteria** — what “done” looks like (optional but recommended)

Save the issue. Leave it **Open** so contributors can find it.

**Result**: The project has at least one open issue; contributors can discover it (if the project is public) and submit contributions.

## Step 5: Verify

- On the institution page, confirm the governance contract and legitimacy chain are visible.
- On the project page, confirm the issue appears in the issue list and is open.
- If the project is public, use **Public Square** or discovery to find your project and see how it looks to others.

## What You've Learned

- You created a digital institution from a template and confirmed its governance.
- You created a project under that institution and added an open issue.
- The institution’s decisions are recorded in the legitimacy chain; the project is ready for contributions.
