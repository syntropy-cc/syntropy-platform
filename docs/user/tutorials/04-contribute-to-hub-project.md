# Tutorial: Contribute to a Hub Project

> **What you'll learn**: Find a project and issue, work on it in the embedded IDE, and submit a contribution (artifact) for review.
>
> **Prerequisites**: Signed-in user; basic familiarity with the Hub. Completing a Learn fragment first helps (you’ll have an artifact to contribute or the skills to build one).
>
> **Time**: ~30 minutes

## Overview

You will discover a project (via the Public Square or a direct link), open an issue, use the integrated IDE to build or adjust an artifact, and submit that artifact as a **Contribution**. The maintainer can then review and accept or reject it.

## Before You Begin

- [ ] You are signed in
- [ ] You have access to the Hub and (optionally) have completed at least one Learn fragment so you’re comfortable with the IDE

## Step 1: Find a project and an issue

In the Hub, open **Public Square** or **Discover** and browse projects, or go directly to a project URL.

Open a **project** that accepts contributions and has **open issues**. Pick an issue that matches your skills (e.g. “Add unit tests for module X”).

**Result**: You have one open issue in front of you with title, description, and acceptance criteria.

## Step 2: Open the contribution context (IDE)

On the issue page, use **Contribute** or **Open in IDE** (or equivalent). The platform may create a **Contribution Sandbox** (Hackin dimension) so you can work in an isolated environment.

The embedded IDE opens with the project context (e.g. repo or artifact) so you can edit and run code.

**Result**: You are in the IDE with the right context to address the issue.

## Step 3: Build or modify the artifact

Implement the change described in the issue: write code, add tests, or edit the artifact. Use the IDE’s run/debug tools as needed.

When the work is ready, **save** and ensure the artifact (file or bundle) is in a state you want to submit.

**Result**: You have an artifact that addresses the issue.

## Step 4: Submit the contribution

From the issue or IDE, choose **Submit contribution** (or equivalent). Attach or select the artifact you produced. Add optional notes for the reviewer.

Submit. The platform creates a **Contribution** linked to the issue and your user; status is typically “submitted” or “in review”.

**Result**: The maintainer can see your contribution and review it.

## Step 5: (Optional) Respond to review

If the maintainer leaves feedback or requests changes, you may see comments on the contribution. Update your artifact or add a new version and resubmit or reply as the UI allows.

When the maintainer **accepts** the contribution, it is integrated into the project (per the project’s workflow). Your portfolio is updated (e.g. contribution accepted, XP).

## What You've Learned

- You discovered a project and an issue and opened the IDE in contribution context.
- You built or modified an artifact and submitted it as a contribution.
- You saw how the contribution flows from submitted → in review → accepted (and how your portfolio reflects it).
